# ジユウノコンパス Improvement Decisions

今までの相談で決まった改善方針をまとめる場所です。
新しい案が出たら、このファイルに追記して、`PRODUCT_PRINCIPLES.md` と矛盾しないか確認します。

## 現在の結論

- 別サイトに分ける必要はない。
- ただし、今のまま全機能を同じ重さで見せると散らかる。
- 追加機能より、まずは編集と整理を優先する。
- サイトの核は「働きすぎない暮らしに近づくための、お金・時間・逃げ道の整理」。
- 一言で言うなら「働きすぎないための生活コンパス診断」。
- 公開コピーでは「働き方を軽くする、生活コンパス診断。」を使い、生活を壊さず自由を増やす方向を前面に出す。
- 実態は「FIREサイト」より「生活の余力診断」に近い。
- FIREは主役ではなく、最終目的地のひとつとして脇役に下げる。
- ただし「FIRE」のほうが伝わる場所ではFIREと書く。「長期の自由ライン」のような言い換えは使わない。
- 主役は「30秒診断」と「今やるミッション」。
- FIRE計算、年金、固定費シミュレーター、時短道具は補助として残す。
- Amazonアフィリエイトは悪くないが、世界観を壊さない見せ方に限定する。

## 残す機能

- 30秒診断。
- 詳細入力。
- 月の余力、貯蓄率、急な出費用の貯金。
- 生活防壁、急な出費耐性、仕事のしんどさ。
- 今やるミッション。
- 固定費削減シミュレーター。
- FIRE目標額と将来予測。
- 年金や40歳以降の生活変化の注意。
- 時間を取り戻す道具。
- 作者情報への導線。

## 弱める機能・表現

- FIREシミュレーター感を前面に出しすぎること。
- FIRE進捗、FIRE到達年齢、FIRE目標額を入口で強く見せること。
- 商品おすすめ感。
- ゲーム、レベル、ボスなどの表現が強くなりすぎること。
- KPIっぽいUI、ランキング、競争、実績集め。
- 「便利機能全部入り」に見える構成。

## 見せ方の改善案

- サイト説明を「FIREシミュレーター」より「働きすぎないための生活コンパス」に寄せる。
- トップコピー: 「働き方を軽くする、生活コンパス診断。」
- 説明文: 「お金と時間の現在地を見ながら、今の余力と整える順番を確認します。」
- トップで「何のサイトか」より「何の不安を軽くする場所か」を伝える。
- 機能をカテゴリ化する。
- カテゴリ案: 整える、逃げ道を作る、時間を取り戻す。
- 「おすすめ」を「時間を取り戻す道具」に言い換える。
- 家電やサービスは「おすすめランキング」ではなく「時間を取り戻したもの」として見せる。
- 「診断とミッション」を最初の主役にする。
- 詳細な数字や将来予測は、必要な人だけ開ける位置に置く。
- 月収は「手取り」と明記する。詳細入力では生活費とローン・保険料の二重入力を避ける説明を入れる。
- 結果画面では同じ注意書きを近い場所で繰り返さない。
- 「ふつう」など単語だけのバッジは避け、「種類」「手間」「状態」がわかる表示にする。
- 余裕がある人にも自然に読めるよう、「このままだと」「でも、こうすると」のような不安寄りの固定見出しは中立にする。
- 作者の思想や本人感を、小さく自然に出す。
- 「なぜ作ったか」「なぜ余力を重視するか」「なぜ働きすぎない暮らしを目指すか」を短く入れる。

## 収益化方針

- 今の段階で広告を増やしすぎない。
- Amazonリンクは「時間を取り戻す道具」として扱う。
- 相性が良い候補: note、Kindle、小さい有料ツール、テンプレ、ライフ設計シート、考え方系コンテンツ、愛用品紹介。
- モデルは大量PV広告より、共感ベースの小収益を優先する。
- 収益導線は、診断体験より目立たせない。
- 「稼ぐための商品リンク」ではなく「世界観の延長としての道具紹介」にする。

## 優先順位

1. まず削る・しまう・言い換える。
2. トップページの核を明確にする。
3. FIREを脇役に下げ、月の余力・生活防壁・次の行動を主役にする。
4. 機能を「整える / 逃げ道を作る / 時間を取り戻す」に整理する。
5. 商品リンクの見せ方を「時間を取り戻す道具」に変更する。
6. 作者の小さな説明を追加する。
7. その後に、必要な機能だけ追加する。

## 今後止めるべき案

- ランキング機能。
- 派手な実績システム。
- ガチャ、装備、強いゲーム化。
- ソーシャル機能やギルド機能。
- 銀行API連携など、巨大サービス化に寄る機能。
- 広告を増やして世界観を壊す収益化。
- 便利そうだが、働きすぎない暮らしと関係が薄い機能。
